AeroSense — Edge-AI Powered Inhaler Usage Monitor
Full Application Summary
1. Project Overview
A full-stack health monitoring web application for asthma patients that tracks inhaler usage in real time, runs Edge AI analysis to detect dangerous patterns, generates smart alerts, and provides an LLM-powered AI health assistant (Groq) for personalised patient guidance.

4. Database Schema
inhaler_usage table
id          INTEGER  PRIMARY KEY AUTOINCREMENT
strength    TEXT     "Low" | "Medium" | "High"
alert       INTEGER  0 or 1 (boolean)
alert_type  TEXT     "low_strength" | "rapid_usage" | "high_usage" | "attack_risk" | null
risk_score  INTEGER  0–100 (AI computed)
severity    TEXT     "none" | "low" | "medium" | "high"
created_at  TEXT     ISO datetime (indexed)
alerts table
id          INTEGER  PRIMARY KEY AUTOINCREMENT
type        TEXT     "low_strength" | "rapid_usage" | "high_usage" | "attack_risk"
message     TEXT     Human-readable alert message
severity    TEXT     "low" | "medium" | "high"
risk_score  INTEGER  0–100
created_at  TEXT     ISO datetime (indexed)
Indexes: idx_inhaler_usage_created_at, idx_inhaler_usage_alert_type, idx_alerts_created_at, idx_alerts_type

6. Edge AI Engine (Core Logic)
Located in 
inhaler.ts

This system simulates Edge AI locally using rule-based inference.
In real-world deployment, this would be replaced by a TensorFlow Lite
model running on the inhaler device (ESP32 or similar microcontroller).
Sensor Simulation
simulateInhalationStrength():
  20% → Low    (PIFR < 30 L/min — poor technique)
  50% → Medium (PIFR 30–60 L/min — normal)
  30% → High   (PIFR > 60 L/min — strong)
Risk Scoring Model (0–100)
rapidScore   = min(usages_in_last_10min × 10, 40)
dailyScore   = max(0, (usages_today - 2) × 5)
strengthScore = Low → +30 | Medium → 0 | High → -5
riskScore    = clamp(rapidScore + dailyScore + strengthScore, 0, 100)
Alert Rules (priority order)
Condition	Alert Type	Severity	Message
riskScore ≥ 75 AND ≥2 recent uses	attack_risk	HIGH	Potential Asthma Attack Warning
≥3 uses in last 10 min	rapid_usage	HIGH	Rapid usage detected
Strength = Low	low_strength	LOW/MEDIUM	Improper inhalation technique
≥6 uses today	high_usage	MEDIUM	High daily usage
Smart Status (Dashboard pill)
"Safe"      → Green  — riskScore < 50, alerts = 0, uses < 5
"Moderate"  → Yellow — riskScore 50–74 OR alerts 1–2 OR uses 5–7
"High Risk" → Red    — riskScore ≥ 75 OR alerts ≥ 3 OR uses ≥ 8
7. Groq AI Health Assistant
Located in 
ai.ts

Model: llama-3.3-70b-versatile via Groq API
Key: Stored in .env as GROQ_API_KEY — never exposed to the browser
Context injection: Before every LLM call, the backend fetches live patient data and injects it into the system prompt:
Uses today, last usage time
Alerts triggered today + types
7-day average daily usage
Current risk score
Frontend: Floating chat button (bottom-right, all pages) with conversation history, typing indicator, suggested questions, and unread badge

10. Complete Data Flow
Patient presses "Use Inhaler"
        │
        ▼
POST /api/use-inhaler
        │
        ├─ simulateInhalationStrength() → "Low" / "Medium" / "High"
        │
        ├─ edgeAIAnalyzer(strength)
        │     ├─ DB: count usages in last 10 min
        │     ├─ DB: count usages today
        │     ├─ Compute: rapidScore + dailyScore + strengthScore
        │     ├─ Clamp to riskScore [0–100]
        │     └─ Apply alert rules → { alert, alertType, message, severity }
        │
        ├─ INSERT inhaler_usage (strength, alert, alertType, riskScore, severity)
        ├─ if alert → INSERT alerts (type, message, severity, riskScore)
        │
        └─ Return JSON to frontend
                │
                ▼
        React Query invalidates 6 queries simultaneously
                │
                ├─ Stats refresh → status pill + risk bar update
                ├─ Usage logs refresh → recent activity updates
                ├─ Alerts refresh → alerts page updates
                ├─ Unread count refresh → nav badge updates
                ├─ Chart refresh → hourly bar chart updates
                └─ Insights refresh → AI insights panel updates
                        │
                        ▼
                Toast notification shown:
                  ✓ Dose Recorded (safe)
                  ⚠️ Alert Triggered (warning)
                  🚨 CRITICAL ALERT (attack_risk)

Patient asks AI assistant a question
        │
        ▼
POST /api/ai/chat { messages: [...] }
        │
        ├─ fetchPatientContext() — live DB data
        ├─ buildSystemPrompt(context) — personalised system message
        ├─ groq.chat.completions.create(llama-3.3-70b-versatile)
        └─ Return { reply: "..." } to front
